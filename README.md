# Hibinka51: Система автоматизации пассажирских перевозок

Модульная платформа для управления логистикой, бронирования поездок и администрирования рейсов.

## 🛠 Стек технологий

- **Backend:** Python 3.11, FastAPI, SQLAlchemy (Async), PostgreSQL.
- **Frontend:** React, TypeScript, Vite.
- **Инфраструктура:** Docker Compose, VS Code Dev Containers, Makefile.

---

## Руководство по развертыванию

Проект использует **VS Code Dev Containers** для создания изолированной среды и **Makefile** для строгого контроля над процессами сборки и запуска. Это полностью исключает ошибки «холодного старта» и конфликты версий.

### Шаг 1: Подготовка конфигурации

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/gakombarov/hibinka-app.git
   cd hibinka-app
   ```
2. Создайте локальный файл переменных окружения из предоставленного шаблона:
   ```bash
   cp .env.example .env
   ```

### Шаг 2: Запуск изолированной среды

1. Откройте папку проекта в редакторе Visual Studio Code.

2. В правом нижнем углу появится уведомление — нажмите "Reopen in Container".

- Альтернатива: нажмите `Ctrl+Shift+P` и введите команду `Dev Containers: Reopen in Container`.

3. Дождитесь завершения сборки контейнера (система скачает образы базы данных и подготовит виртуальное окружение Python).

### Шаг 3: Первичная установка зависимостей (Выполняется 1 раз)

Чтобы избежать состояния гонки при первом запуске, зависимости фронтенда устанавливаются вручную.

- Откройте терминал в VS Code (Ctrl + ~) и выполните команду:

  ```bash
  make install
  Дождитесь сообщения об успешной установке пакетов npm.
  ```

### Шаг 4: Запуск сервисов

Вы можете запустить все сервисы одной кнопкой через встроенные задачи VS Code:

Нажмите `Ctrl + Shift + P`.

Введите `Tasks: Run Task`.

Выберите \*🚀 ЗАПУСТИТЬ ПРОЕКТ.

Система автоматически откроет три терминала, очистит зависшие порты (если они были) и запустит Backend, Landing и Dashboard.

### 🔗 Доступные сервисы

- **8000**: Backend API (Swagger)
- **5173**: Landing Page (Основной интерфейс)
- **5174**: Shared Sandbox (Тестирование компонентов)

## Реализация требований к проекту

---

### ⚛️ 1. Фреймворк React

- **Современный подход:** Использованы функциональные компоненты и хуки (`useState`, `useEffect`, `useMemo`, `useRef`).
- **Композиция:** Продуманная архитектура компонентов (выделены `Layout`, изолированные UI-элементы, карточки рейсов).
- **Темизация:** Реализована кастомная поддержка **Dark/Light mode** через `ThemeProvider` с сохранением выбора пользователя в `localStorage`.

### 🛣️ 2. Маршрутизация (React Router v6)

- **Базовый роутинг:** Четкое разделение на публичную зону (`/`) и защищенную панель администратора (`/dashboard`).
- **Вложенные роуты:** Использование `<Outlet />` для отрисовки контента внутри `DashboardLayout`.
- **Динамические пути:** Обработка маршрутов вида `/dashboard/bookings/:id` (реализовано через хук `useParams`).
- **UX/UI:** Стилизация активных ссылок в навигации с использованием свойства `isActive`.

### ⚡ 3. Стейт-менеджер Redux Toolkit

- **Глобальное состояние:** Создан централизованный `store`, логика авторизации вынесена в `authSlice`.
- **Персистентность:** Данные сессии и токены синхронизируются с `LocalStorage` (пользователь не теряет сессию при обновлении страницы `F5`).
- **Hooks:** Взаимодействие со стором через типизированные хуки `useSelector` и `useDispatch`.

### 🔄 4. Работа с API

- **Клиент:** Настроен API-клиент на базе `axios` для стабильного взаимодействия с FastAPI бэкендом.
- **Функционал:** Загрузка списка заявок (`GET`), обработка состояний загрузки (`CircularProgress`).
- **Обработка ошибок:** Использование блоков `try/catch` с визуальными уведомлениями через `Snackbar` и `Alert`.
- **Фильтрация:** Реализован быстрый поиск по списку заявок на стороне клиента.

### 📝 5. Работа с элементами форм

- **Сложные формы:** Реализован компонент `BookingForm.tsx` для оформления заявок.
- **Логика:** Сбор данных, валидация полей и отправка `POST` запроса на сервер.

### 🎨 6. UI-библиотека Material-UI

- **Компоненты:** Интерфейс построен на базе `Grid`, `Box`, `Typography`, `Table`, `Drawer`, `TextField`, `Modal`.
- **Дизайн-система:** Использована кастомная палитра и стилизация через `createTheme` и систему `sx`.

### 📖 7. Storybook для UI-Kit

- **Документация:** Настроена изолированная среда для разработки и тестирования компонентов.
- **Stories:** Добавлены файлы `.stories.tsx` для переиспользуемых элементов (например, кнопка `Button`).
- **Команды:** Запуск через `npm run storybook`.

### 🛠️ 8. Инструменты разработки

- **Качество кода:** Настроены `ESLint` для статического анализа и `Prettier` для единообразного форматирования.
- **Типизация:** Весь проект написан на `TypeScript`, что гарантирует безопасность типов.

---

### 🛠 Управление

- `make install` — установка зависимостей.
- `make start-server` — запуск серверной части
- `make start-client` — запуск клиентской части
