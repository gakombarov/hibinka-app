# Hibinka51: Система автоматизации пассажирских перевозок

Модульная платформа для управления логистикой, бронирования поездок и администрирования рейсов. Проект включает клиентскую витрину для бронирования и защищенную панель администратора для обработки заявок.

## 🛠 Технологический стек

- **Backend:** Python 3.11, FastAPI, SQLAlchemy (Async), PostgreSQL.
- **Frontend:** React, TypeScript, Vite, Redux Toolkit, React Router v6.
- **UI-библиотека:** Material-UI (MUI) + Storybook.
- **Инфраструктура:** Docker Compose, VS Code Dev Containers, Makefile.

---

## 🚀 Руководство по развертыванию

Проект использует **VS Code Dev Containers** и **Makefile** для полной изоляции среды разработки. Это исключает конфликты версий и ошибки «холодного старта».

### Шаг 1: Подготовка конфигурации

1. Склонируйте репозиторий:
   git clone https://github.com/gakombarov/hibinka-app.git
   cd hibinka-app
2. Создайте файл переменных окружения из шаблона:
   cp .env.example .env

### Шаг 2: Запуск изолированной среды

1. Откройте папку проекта в VS Code.
2. Нажмите **Ctrl+Shift+P** и выберите: `Dev Containers: Reopen in Container`.
3. Дождитесь завершения сборки контейнера.

### Шаг 3: Установка зависимостей

В терминале VS Code (внутри контейнера) выполните:
make install

### Шаг 4: Запуск сервисов

Нажмите **Ctrl + Shift + P** -> `Tasks: Run Task` -> **🚀 ЗАПУСТИТЬ ПРОЕКТ**.

- **Порт 8000**: Backend API (Swagger)
- **Порт 5173**: Landing Page (Клиентская часть)
- **Порт 5174**: Shared Sandbox (Storybook / Тестирование компонентов)

## 🧪 Инструкция по проверке работоспособности

Чтобы проверить полную цепочку работы системы, выполните следующие действия:

### 1. Создание администратора

Для доступа в защищенную панель управления необходимо создать пользователя. В терминале бэкенда (внутри контейнера) запустите скрипт:
`python create_user.py`
_Используйте логин и пароль, прописанные в этом скрипте, для входа в систему._

### 2. Создание заявок (Landing Page)

1. Откройте **Landing Page** (порт 5173).
2. Заполните форму в разделе **"Заказать трансфер"** (BookingForm.tsx).
3. После отправки заявка автоматически попадет в базу данных и станет доступна для обработки.

### 3. Авторизация и управление (Dashboard)

1. В шапке лендинга нажмите кнопку **"Войти"** и авторизуйтесь (LoginForm.tsx).
2. Система перенаправит вас в **Dashboard** (Журнал поездок).
3. Все созданные на лендинге заявки отобразятся в общем списке.
4. Чтобы перейти к управлению конкретной заявкой, нажмите кнопку **"Управление"** в карточке рейса.

## 🎓 Реализация требований (Roadmap для проверки)

### ⚛️ 1. Фреймворк React

- **Хуки (Hooks):** Использование `useState`, `useEffect`, `useMemo` (в теме) и `useRef`.
- **Композиция:** Выделены Layout-компоненты и изолированные UI-элементы.
- **Где смотреть:** `src/App.tsx`, `src/pages/LandingPage.tsx`, `src/shared/components/ui/Button.tsx`.

### 🛣️ 2. Маршрутизация (React Router v6)

- **Конфигурация:** Реализован базовый роутинг и защищенные пути.
- **Динамика:** Обработка параметров `/dashboard/bookings/:id` через `useParams`.
- **UX:** Подсветка активной ссылки в меню через `location.pathname`.
- **Где смотреть:** `src/App.tsx`, `src/components/layout/DashboardLayout.tsx`.

### ⚡ 3. Стейт-менеджер Redux Toolkit

- **Global Store:** Логика авторизации вынесена в `authSlice`.
- **LocalStorage:** Синхронизация токена и состояния пользователя.
- **Где смотреть:** `src/store/store.ts`, `src/store/authSlice.ts`.

### 🔄 4. Работа с API

- **Интеграция:** Клиент на базе Axios с обработкой Loading/Error состояний.
- **Поиск:** Реализована фильтрация данных на стороне клиента.
- **Где смотреть:** `src/api/client.ts`, `src/features/bookings/BookingsList.tsx`.

### 📝 5. Работа с формами

- **Валидация:** Сложные формы бронирования и авторизации на базе `react-hook-form`.
- **Где смотреть:** `src/shared/components/ui/BookingForm.tsx`, `src/features/auth/LoginForm.tsx`.

### 🎨 6. UI-библиотека Material-UI

- **MUI Theme:** Кастомная настройка темы (цвета, шрифты, скругления).
- **Компоненты:** Grid, Box, Drawer, Modal, Snackbar, Иконки.
- **Где смотреть:** `src/shared/theme/theme.ts`, `src/shared/components/cards/AdminTripCard.tsx`.

### 📖 7. Storybook для UI-Kit

- **Изолированная среда:** Настроены стори для переиспользуемых компонентов.
- **Где смотреть:** `src/shared/components/ui/Button.stories.tsx`.

### 🛠️ 8. Инструменты разработки

- **Типизация:** Весь проект написан на TypeScript.
- **Линтинг:** Настроены ESLint и Prettier.
